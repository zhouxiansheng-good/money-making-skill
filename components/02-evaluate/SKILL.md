# 02-evaluate：评估层

> **"集中优势兵力"** — 只打有把握的仗，不分散精力。

## 职责

系统化评估赚钱机会，输出Go/NoGo决策。核心原则：**只做非对称回报的事，只做匹配特定知识的事，绝不触碰红线**。

## 前置条件

- 思维层（01-think）已完成特定知识画像和杠杆识别
- 用户的红线约束已明确

## 组件

### 1. 机会扫描器

**目标**：从信息中识别赚钱机会。

**扫描维度**：

| 维度 | 扫描问题 | 信号 |
|------|----------|------|
| 痛点信号 | 有人在抱怨什么？ | 论坛/社群/评论区的吐槽 |
| 趋势信号 | 什么正在增长？ | 搜索量/下载量/用户数上升 |
| 政策信号 | 政策在鼓励什么？ | 补贴/减税/试点 |
| 技术信号 | 新技术降低了什么门槛？ | AI/自动化/新工具 |
| 竞争信号 | 竞争对手在做什么？ | 新产品/新服务/新市场 |

**机会描述模板**：

```yaml
opportunity:
  name: "机会名称"
  description: "一句话描述"
  pain_point: "解决的痛点"
  target_user: "目标用户"
  revenue_model: "怎么赚钱"
  leverage_type: "代码/媒体/资本/劳动力"
  estimated_asymmetric_ratio: "收益:风险比估计"
```

### 2. 非对称回报检查

**目标**：确保只做"赢了赚很多、输了亏很少"的事。

**检查流程**：

```
输入: 机会描述
  ↓
Step 1: 估算最佳情况收益
  - 如果一切顺利，最多能赚多少？（月收入）
  - 这个收益能持续多久？（一次性/月度/年度）

Step 2: 估算最差情况损失
  - 如果完全失败，最多亏多少？（时间+金钱）
  - 失败后能收回什么？（经验/技能/关系）

Step 3: 计算非对称比
  asymmetric_ratio = 最佳收益 / 最差损失

Step 4: 判定
  → ≥ 10:1 → 强Go（非对称回报极佳）
  → 3:1~10:1 → Go（非对称回报可接受）
  → 1:1~3:1 → NeedMoreInfo（需要优化方案提升回报比）
  → < 1:1 → NoGo（风险大于收益）
```

**关键原则**：潜在收益至少是损失的3倍，最好是10倍。

### 3. 特定知识匹配

**目标**：检查机会是否匹配你的特定知识。

**匹配流程**：

```
输入: 机会描述 + 你的特定知识画像
  ↓
Step 1: 列出机会所需的核心能力
Step 2: 与你的特定知识逐项比对
Step 3: 计算匹配度
  matching_score = 匹配项数 / 所需核心能力数 × 100%

Step 4: 判定
  → ≥ 80% → 强匹配（你的主场）
  → 60%~80% → 可匹配（需要补充少量能力）
  → 40%~60% → 弱匹配（需要大量学习）
  → < 40% → 不匹配（不是你的赛道）
```

**关键原则**：只做匹配度≥60%的机会。低于60%说明这不是你的赛道。

### 4. 红线过滤器

**目标**：过滤不符合硬约束的机会。

**红线检查清单**：

| 红线 | 检查问题 | 通过标准 | 不通过处理 |
|------|----------|----------|-----------|
| 合法合规 | 是否涉及法律灰色地带？ | 完全合法 | NoGo |
| 时间可控 | 每天投入是否超过2小时？ | ≤2小时/天 | NoGo或缩小范围 |
| 资金可控 | 启动资金是否超过500元？ | ≤500元 | NoGo或寻找低成本替代 |
| 非纯卖劳 | 是否纯时间换钱？ | 有可规模化空间 | NoGo |

**红线一票否决**：任一红线不通过 → 直接NoGo，不进入后续评估。

## 综合评估输出

```yaml
decision: "Go" | "NoGo" | "NeedMoreInfo"
opportunity: "机会名称"
leverage_type: "代码/媒体/资本/劳动力/组合"
leverage_score: 1-10
asymmetric_ratio: "收益:风险比"
specific_knowledge_match: 0-100%
red_line_check:
  legal: true/false
  time_ok: true/false
  capital_ok: true/false
  not_labor: true/false
reason: "决策理由"
action_path: "建议行动路径（Go时）"
improvement_suggestion: "改进建议（NeedMoreInfo时）"
```

## 路由

- 决策为Go → 进入 `components/03-leverage/SKILL.md`
- 决策为NoGo → 终止，告知原因
- 决策为NeedMoreInfo → 回到机会扫描器补充信息
