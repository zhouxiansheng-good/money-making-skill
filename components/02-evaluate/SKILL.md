# 02-evaluate：评估层

> **"集中优势兵力"** — 只打有把握的仗，不分散精力。

## 职责

系统化评估赚钱机会，输出Go/NoGo决策。核心原则：**只做非对称回报的事，只做匹配特定知识的事，绝不触碰红线，必须验证市场可行性**。

## 前置条件

- 思维层（01-think）已完成特定知识画像和杠杆识别
- 用户的红线约束已明确

## 组件

### 1. 机会扫描器

**目标**：从信息中识别赚钱机会。

**扫描维度**：

| 维度 | 扫描问题 | 信号 |
|------|----------|------|
| 痛点信号 | 有人在抱怨什么？ | 论坛/社群/评论区的吐槽 |
| 趋势信号 | 什么正在增长？ | 搜索量/下载量/用户数上升 |
| 政策信号 | 政策在鼓励什么？ | 补贴/减税/试点 |
| 技术信号 | 新技术降低了什么门槛？ | AI/自动化/新工具 |
| 竞争信号 | 竞争对手在做什么？ | 新产品/新服务/新市场 |

**机会描述模板**：

```yaml
opportunity:
  name: "机会名称"
  description: "一句话描述"
  pain_point: "解决的痛点"
  target_user: "目标用户"
  revenue_model: "怎么赚钱"
  leverage_type: "代码/媒体/资本/劳动力"
  estimated_asymmetric_ratio: "收益:风险比估计"
```

### 2. 非对称回报检查

**目标**：确保只做"赢了赚很多、输了亏很少"的事。

**检查流程**：

```
输入: 机会描述
  ↓
Step 1: 估算最佳情况收益
  - 如果一切顺利，最多能赚多少？（月收入）
  - 这个收益能持续多久？（一次性/月度/年度）
  - 数据来源：平台公开数据/同行访谈/历史案例（必须标注）

Step 2: 估算最差情况损失
  - 如果完全失败，最多亏多少？（时间+金钱+机会成本）
  - 失败后能收回什么？（经验/技能/关系）
  - 是否有不可逆损失？（法律记录、信用污点、账号封禁）

Step 3: 计算非对称比
  asymmetric_ratio = 最佳收益 / 最差损失

Step 4: 判定
  → ≥ 10:1 → 强Go（非对称回报极佳）
  → 3:1~10:1 → Go（非对称回报可接受）
  → 1:1~3:1 → NeedMoreInfo（需要优化方案提升回报比）
  → < 1:1 → NoGo（风险大于收益）
```

**关键原则**：潜在收益至少是损失的3倍，最好是10倍。收益估算必须标注数据来源（实测/同行/估算）。

### 3. 特定知识匹配

**目标**：检查机会是否匹配你的特定知识。

**匹配流程**：

```
输入: 机会描述 + 你的特定知识画像
  ↓
Step 1: 列出机会所需的核心能力（至少5项）
Step 2: 与你的特定知识逐项比对
Step 3: 计算匹配度
  matching_score = 匹配项数 / 所需核心能力数 × 100%

Step 4: 判定
  → ≥ 80% → 强匹配（你的主场）
  → 60%~80% → 可匹配（需要补充少量能力）
  → 40%~60% → 弱匹配（需要大量学习）
  → < 40% → 不匹配（不是你的赛道）
```

**关键原则**：只做匹配度≥60%的机会。低于60%说明这不是你的赛道。

### 4. 法律风险审查器（新增）

**目标**：系统化识别法律风险，避免踩坑。

**六维法律风险矩阵**：

| 维度 | 检查问题 | 风险等级 | 通过标准 | 不通过处理 |
|------|----------|----------|----------|-----------|
| **知识产权** | 是否涉及抄袭、商标侵权、字体/图片/音乐版权？使用的AI生成内容是否有版权争议？ | 🔴高/🟠中/🟢低 | 所有素材有合法来源或自制 | NoGo或替换素材 |
| **税务合规** | 收入是否需要报税？是否需要注册个体户/公司？是否涉及增值税/个人所得税？ | 🔴高/🟠中/🟢低 | 了解纳税义务，有合规计划 | NoGo或咨询会计 |
| **平台规则** | 目标平台（闲鱼/淘宝/抖音/小红书等）是否允许该业务？是否有限流/封号风险？ | 🔴高/🟠中/🟢低 | 平台规则明确允许 | NoGo或换平台 |
| **数据合规** | 是否收集/处理用户数据？是否符合《个人信息保护法》？ | 🔴高/🟠中/🟢低 | 不收集或合规收集 | NoGo或整改 |
| **广告法** | 宣传用语是否涉及虚假宣传、极限词（最、第一、唯一等）？ | 🟠中/🟢低 | 无虚假宣传，无极限词 | 修改文案 |
| **特殊许可** | 是否涉及食品、医疗、教育、金融等需要许可证的行业？ | 🔴高 | 有相应许可证或不涉及 | NoGo |

**法律风险审查流程**：

```
输入: 机会描述 + 业务模式
  ↓
Step 1: 识别业务类型
  - 内容创作？电商？服务？软件？培训？
  
Step 2: 六维逐项检查
  - 对每个维度回答：是否涉及？风险等级？如何规避？
  
Step 3: 综合判定
  - 任一维度为🔴高且无法规避 → 直接NoGo
  - ≥2个维度为🟠中 → 需整改后重新评估
  - 全部🟢低 → 通过
  
Step 4: 输出法律风险评估报告
  - 各维度风险等级
  - 具体规避建议
  - 是否需要专业法律咨询
```

**常见法律风险速查**：

| 业务类型 | 常见风险 | 规避建议 |
|----------|----------|----------|
| AI内容创作 | AI生成内容版权归属不明 | 明确平台规则，人工大幅修改，保留创作证据 |
| 电商/闲鱼 | 假货、侵权、虚假宣传 | 不卖品牌货，实拍图，真实描述 |
| 知识付费/培训 | 无证办学、虚假宣传 | 不承诺结果，明确免责声明 |
| 数据/爬虫服务 | 非法获取数据、侵犯隐私 | 只处理公开数据，不碰个人信息 |
| 自媒体/带货 | 极限词、虚假宣传、侵权 | 审核文案，使用正版素材 |

### 5. 市场可行性验证器（新增）

**目标**：验证机会在真实市场中是否可行。

**验证三维度**：

#### 5.1 竞品分析

```
Step 1: 找出直接竞品（至少3个）
  - 谁在提供类似服务/产品？
  - 他们的定价是多少？
  - 他们的优势和劣势是什么？

Step 2: 分析竞争强度
  - 市场是否已饱和？
  - 进入壁垒高还是低？
  - 差异化空间有多大？

Step 3: 判定
  → 竞争少+差异化空间大 → 强Go
  → 竞争中等+有差异化空间 → Go
  → 竞争饱和+无明显差异化 → NeedMoreInfo或NoGo
```

#### 5.2 需求验证

```
Step 1: 验证需求存在性
  - 有人在搜索相关关键词吗？（百度指数/微信指数/5118）
  - 相关社群/论坛有人讨论吗？
  - 竞品有销量/有客户吗？

Step 2: 验证需求强度
  - 用户愿意为此付费吗？
  - 付费意愿有多强？（刚需/改善型/可有可无）

Step 3: 判定
  → 有搜索量+有讨论+有付费 → 强Go
  → 有信号但不够强 → Go（需MVP验证）
  → 无明显信号 → NeedMoreInfo
```

#### 5.3 平台政策扫描

```
Step 1: 识别目标平台
  - 主要运营平台：_____
  - 辅助分发平台：_____

Step 2: 扫描平台政策
  - 该平台是否允许此类内容/服务？
  - 平台对AI生成内容的态度？（鼓励/中立/限制/禁止）
  - 平台流量分配机制是否利好新入局者？
  - 近期是否有政策变动趋势？

Step 3: 判定
  → 平台鼓励+流量机制友好 → 强Go
  → 平台允许+流量机制中性 → Go
  → 平台限制或流量机制不利 → NoGo或换平台
```

### 6. 红线过滤器

**目标**：过滤不符合硬约束的机会。

**红线检查清单**：

| 红线 | 检查问题 | 通过标准 | 不通过处理 |
|------|----------|----------|-----------|
| 合法合规 | 法律风险审查是否通过？ | 六维矩阵无🔴高风险 | NoGo |
| 市场可行 | 市场可行性验证是否通过？ | 竞品+需求+平台至少两项通过 | NoGo或调整方向 |
| 时间可控 | 每天投入是否超过2小时？ | ≤2小时/天 | NoGo或缩小范围 |
| 资金可控 | 启动资金是否超过500元？ | ≤500元 | NoGo或寻找低成本替代 |
| 非纯卖劳 | 是否纯时间换钱？ | 有可规模化空间 | NoGo |

**红线一票否决**：任一红线不通过 → 直接NoGo，不进入后续评估。

### 7. 二阶思维检查（新增）

**目标**：分析连锁反应，避免短视决策。

**检查流程**：

```
Step 1: 如果成功了，然后呢？
  - 规模扩大后会有什么新问题？（供应链、客服、合规）
  - 成功会吸引多少竞争者？
  - 平台政策会因此改变吗？

Step 2: 如果失败了，然后呢？
  - 最坏情况是什么？（法律、信用、财务）
  - 失败后的退出成本？
  - 失败后还能转向什么方向？

Step 3: 6个月后回头看
  - 这个决策在6个月后还会觉得对吗？
  - 6个月后市场会变成什么样？
```
